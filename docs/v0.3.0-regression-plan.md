## Architecture Notes

- Settings are stored in `chrome.storage.local` under `linkedinHubspotAiAssistant.settings`.
- Existing v0.2 ICP fields live in `UserSettings`.
- Sidebar analysis sends `{ profile, userSettings }` to `/ai/analyze-profile`.
- DM generation sends `{ profile, analysis, messageType, userSettings }` to `/ai/generate-dm`.
- API validates payloads with shared zod schemas and compacts LinkedIn profile text before parsing.
- Lead scoring currently blends OpenAI output with deterministic heuristic scoring.
- Sidebar render safety depends on `normalizeAnalysisResult` and `SidebarErrorBoundary`.
- The current `Edit ICP` button calls Options APIs directly from the content-script sidebar and falls back to a constructed extension URL, which can produce a blocked Chrome page.

## Regression Risks Reviewed

- License activation: storage changes must not alter `LICENSE_STATE_KEY`.
- ICP storage: v0.2 fields must remain unchanged and merge with Seller Context defaults.
- Sidebar mount: no code may remove the ErrorBoundary or content-script root preservation.
- Analyze Profile: request payload must remain compact and validated.
- visibleTextSample compaction: shared profile limits must remain enforced.
- Score calculation: final score must be bounded and evidence must not invent hidden facts.
- AI response parsing: old/malformed arrays must normalize safely.
- DM rendering: v0.3 optional metadata must not break v0.2 variants.
- Options navigation: Options opening should happen via background runtime message.
- HubSpot contact: real-name preflight and `hs_linkedin_url` mapping must remain.
- HubSpot note: note must skip empty fields and escape HTML.
- Follow-up task: task payload shape must remain unchanged.
- Manifest permissions: no new broad permissions should be added.
- Chrome Web Store package: ZIP must contain only production dist output.

## Manual/E2E Notes

The repository does not currently define a Playwright or Chrome-extension E2E command. v0.3.0 will add deterministic unit/component coverage for the extension UI and background message flow. Real Chrome extension loading, real LinkedIn pages, and real HubSpot/Stripe calls remain manual QA steps unless a browser extension harness is added later.
